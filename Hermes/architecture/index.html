Hermes Architecture

Last updated: 2026-06-06

Hermes is the system that handles correspondence between recruiters and me. I built it because the volume of recruiter outreach made manual triage impractical and the alternative (ignoring it) cost me opportunities I would have wanted to look at. This page explains how it works end to end so you can know exactly what you are interacting with.

The two ways a conversation starts

1. You email me directly at recruit@martymcenroe.ai

Your message lands in a queue I call Held for Initial Review. Hermes does not auto-engage. I see your message in a dashboard, read it, and either release it to the AI pipeline (which sends the first email described below) or drop it. Until I release, nothing goes out.

2. I forward a thread from my own inbox

When I forward a recruiter thread from one of my personal addresses to my Hermes forwarder address, Hermes treats my forward as my explicit choice to engage. The first email goes out automatically in response.

In both paths, the first email is sent only after my judgement. New cold inbound never auto-engages.

The first email is a program, not AI

The first email is sent by software I wrote. The body is a template I composed myself, stored in a source file. No language model generates it. The only variable is your first name in the greeting, if I have it. The resume attachments are the same for every recipient.

People are familiar with programs. Email autoresponders, scheduled newsletters, ticketing systems. The first email is in that category. It is a deterministic process, not AI.

The template asks you to star a public GitHub repository before further conversation. There are two reasons:

  • The star tells me you are a real person. GitHub's captcha is non-trivial. Bots cannot pass it reliably. If you star, I have a strong signal you are human.
  • The star tells me you have seen my work. Starring requires you to land on the repository page. If you are going to represent me to anyone, I would rather you have looked at what I have built.

The resume is attached to the first email unconditionally. You do not need to star to receive it. The star is a filter for continued engagement, not a paywall for the resume.

Replies after the first email use AI

If you reply, your message goes back to Hermes and a response is generated using a large language model. I designed the system. I wrote the prompts. I set the rules the system follows and the gates it has to pass before it sends anything.

Every generated reply passes through automated quality gates that check for safety issues, hostile tone, and false claims about repository stars. Replies that fail the gates are routed to me for review before sending. Replies that pass are sent automatically.

Specific model and infrastructure details are in the Tech Stack section below.

Tech stack

Hermes is built on:

  • Cloudflare Workers for the inbound email handler and the dashboard backend.
  • Cloudflare D1 (SQLite) for conversation state.
  • Cloudflare R2 for resume attachments and asset caching.
  • Cloudflare Vectorize for retrieval-augmented context.
  • AWS Lambda for the asynchronous AI generation pipeline.
  • AWS SQS as the queue between the Worker and the Lambda.
  • AWS SES for outbound email.
  • AWS Bedrock + Claude (Anthropic) for AI generation.
